## 1.1 计算机网络概述

### 1.1.1 计算机网络的定义

计算机网络是将**不同地理位置**的、具有**独立功能**的多个计算机系统，通过**通信设备和线路**连接起来，在网络**协议**的控制和网络**操作系统**的管理下，实现**资源共享**和信息传递的系统。

**关键词**：自主计算机、互联、资源共享、协议

### 1.1.2 计算机网络的组成

| 划分方式      | 组成部分         | 说明                                         |
| --------- | ------------ | ------------------------------------------ |
| **按物理组件** | 硬件 + 软件 + 协议 | 硬件：主机/端系统、通信链路、交换设备；软件：应用、操作系统；协议：规则集合     |
| **按逻辑功能** | 通信子网 + 资源子网  | 通信子网：传输介质、通信设备、相关协议，负责数据传输；资源子网：主机、终端、软件资源 |
| **按工作方式** | 边缘部分 + 核心部分  | 边缘部分：端系统(主机)；核心部分：路由器、交换机等组成的网络            |

### 1.1.3 计算机网络的功能

- **数据通信**（最基本功能）：计算机间传送信息
- **资源共享**（核心功能）：硬件、软件、数据资源共享
- **分布式处理**：将大型任务分配给多台计算机协同完成
- **提高可靠性**：冗余备份，某台故障时其他可替代
- **负载均衡**：均衡分配工作负载

### 1.1.4 计算机网络的分类

#### 按分布范围分类

| 类型             | 范围      | 特点              |
| -------------- | ------- | --------------- |
| **个人区域网(PAN)** | 10m 左右  | 蓝牙、红外，个人设备互联    |
| **局域网(LAN)**   | 几米~几千米  | 速率高、误码率低、一个单位所有 |
| **城域网(MAN)**   | 5~50km  | 覆盖城市范围          |
| **广域网(WAN)**   | 几十~几千公里 | 跨越国家/洲际，因特网核心   |

#### 按拓扑结构分类

| 拓扑      | 特点              | 优缺点             |
| ------- | --------------- | --------------- |
| **星形**  | 中央节点，各节点直接连中央节点 | 单点故障影响大，易于管理    |
| **总线形** | 所有节点共享一条总线      | 成本低，但总线故障全网瘫痪   |
| **环形**  | 节点连成闭合环路        | 单向/双向传输，单点断则全网断 |
| **网状形** | 节点间不完全/完全连接     | 可靠性最高，成本最高      |
| **树形**  | 分层结构            | 扩展性好，根节点故障影响大   |

#### 按交换技术分类

| 交换方式 | 原理 | 特点 |
|---------|------|------|
| **电路交换** | 建立专用物理通路 | 实时性强，无时延，线路利用率低 |
| **报文交换** | 存储转发整份报文 | 无呼叫阻塞，时延大 |
| **分组交换** | 存储转发分组（逐段） | 灵活高效，是计算机网络的基础 |

#### 按传输介质分类

- **有线网络**：双绞线、同轴电缆、光纤
- **无线网络**：微波、卫星、红外、无线电

### 1.1.5 计算机网络的性能指标 ——★408高频考点

#### 速率

- **定义**：单位时间内传输的比特数（数据率/比特率）
- **单位**：bps(b/s) → Kbps → Mbps → Gbps → Tbps
- **进率**：$10^3$（与存储容量进率 $2^{10}=1024$ 不同）

#### 带宽

- **频域含义**：信号频率范围，单位 Hz
- **时域/网络含义**：信道最高数据率，单位 bps
- **注意**：计算机网络中"带宽"指**最高数据率**，即速率的上限

#### 吞吐量

- **定义**：单位时间内通过某个网络（或信道）的**实际**数据量
- **特点**：≤ 带宽，受多种因素影响
- **关系**：吞吐量 ≤ 带宽

#### 时延 ——★★★ 408必考计算

> **总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延**

| 时延类型 | 公式/含义 | 影响因素 |
|---------|----------|---------|
| **发送时延** | $$T_{send} = \frac{数据帧长度(bit)}{发送速率(bps)}$$ | 数据帧长、信道带宽 |
| **传播时延** | $$T_{prop} = \frac{信道长度(m)}{信号传播速率(m/s)}$$ | 信道长度、介质类型 |
| **处理时延** | 节点分析首部、查表等 | 节点性能 |
| **排队时延** | 等待输出链路空闲 | 网络拥塞程度 |

> **关键区分**：发送时延发生在**机器内部**（路由器/主机），传播时延发生在**信道**上。

#### 时延带宽积

$$时延带宽积 = 传播时延 \times 带宽$$

- **物理意义**：以比特为单位的链路长度，即链路上**第一比特到达终点时，链路上已发送的比特数**
- 即**管道中可容纳的比特数**

#### 往返时延(RTT)

- **定义**：从发送方发送第一个比特开始，到接收到接收方确认所经过的时间
- **RTT 包含**：往返传播时延 + 中间节点的处理时延 + 排队时延
- **注意**：RTT **不包含**发送时延（发送方发完数据的时间不算在内）
- **有效数据率** = 数据量 / (RTT + 发送时延)

#### 利用率

- **信道利用率** = 有数据通过的时间 / 总时间
- **网络利用率**：全网信道利用率的加权平均
- **D = D₀ / (1 - U)**：$D$ 为实际时延，$D_0$ 为空闲时延，$U$ 为利用率
  - 当 U ≥ 50%：时延急剧增加
  - 当 U → 1：时延 → ∞

#### 丢包率

- **定义**：丢失分组数 / 发送总分组数
- **原因**：路由器缓存满、误码、节点故障
- **反映**：网络拥塞程度



## 1.2 计算机网络体系结构

### 1.2.1 分层结构的基本概念

#### 为什么要分层

- **化繁为简**：将复杂问题分解为若干小问题
- **各层独立**：每层只需关注自己的功能
- **灵活性好**：某层变化不影响其他层
- **易于标准化**：促进不同厂商设备互通

#### 核心术语

| 术语 | 定义 | 说明 |
|------|------|------|
| **实体(Entity)** | 每一层中完成特定功能的硬件或软件进程 | 对等实体：不同机器上**同一层**的实体 |
| **协议(Protocol)** | 对等实体间通信规则的集合 | 三要素：语法、语义、同步(时序) |
| **服务(Service)** | 下层向上层提供的功能（**垂直方向**） | 服务是接口，协议是规则 |
| **服务访问点(SAP)** | 相邻两层交换信息的地点 | 第 n 层 SAP 是第 n+1 层访问第 n 层服务的地方 |
| **协议数据单元(PDU)** | 对等实体间传送的数据单位 | 物理层：比特流；数据链路层：帧；网络层：IP数据报/分组；传输层：TCP/UDP报文段 |
| **服务数据单元(SDU)** | 层与层之间交换的数据单位 | 可被拆分为多个PDU |

> ★★★ **三要素核心**：**语法**——数据格式，**语义**——含义(Action)，**同步/时序**——顺序

### 1.2.2 协议三要素 ——★408常考

| 要素 | 含义 | 举例 |
|------|------|------|
| **语法** | 数据与控制信息的**结构或格式** | 用ASCII编码还是二进制 |
| **语义** | 每种控制信息的**含义**，应做出的响应 | 收到 "ACK" 表示确认 |
| **同步/时序** | 事件实现**顺序**的详细说明 | 先建立连接，再发送数据 |



## 1.3 OSI七层参考模型 ——★★★ 408核心考点

### 1.3.1 各层功能速查表

| 层次  | 名称        | 核心功能               | 数据单位         | 典型协议/设备           |     |
| --- | --------- | ------------------ | ------------ | ----------------- | --- |
| 7   | **应用层**   | 为用户程序提供网络服务接口      | 报文(Message)  | HTTP、FTP、DNS、SMTP |     |
| 6   | **表示层**   | 数据格式转换、加解密、压缩      | 报文           | JPEG、ASCII、加密/解密  |     |
| 5   | **会话层**   | 建立/管理/终止会话，同步      | 报文           | RPC、SQL           |     |
| 4   | **传输层**   | 端到端可靠/不可靠传输、复用分用   | 报文段(Segment) | TCP、UDP           |     |
| 3   | **网络层**   | 路由选择、分组转发、拥塞控制     | 分组(Packet)   | IP、ICMP、路由器       |     |
| 2   | **数据链路层** | 成帧、差错控制、MAC访问、流量控制 | 帧(Frame)     | PPP、以太网、交换机       |     |
| 1   | **物理层**   | 比特流的透明传输           | 比特(Bit)      | 集线器、中继器           |     |

### 1.3.2 各层详解

#### 物理层（第1层）

- 透明传输比特流
- 定义机械、电气、功能、规程四大接口特性
- 数据单位：**比特(Bit)**
- 设备：集线器(Hub)、中继器(Repeater)
- **不能**区分不同帧/分组，只传比特

#### 数据链路层（第2层）

- 将不可靠的物理链路变为**可靠的数据链路**（逻辑上无差错）
- **四大功能**：
  - 封装成帧（添加首部和尾部）
  - 差错控制（CRC检错、重传）
  - 流量控制（防止发送过快淹没接收方）
  - 介质访问控制(MAC)（广播信道中分配使用权）
- 数据单位：**帧(Frame)**
- 设备：交换机(Switch)、网桥(Bridge)
- 在帧头部使用**MAC地址(物理地址)**进行寻址

#### 网络层（第3层）

- 实现主机到主机的通信（点→点）
- **核心功能**：
  - 路由选择（确定分组从源到目的的路径）
  - 分组转发（将分组从输入端口转移到输出端口）
  - 拥塞控制
- 数据单位：**分组(Packet) / IP数据报**
- 使用**IP地址(逻辑地址)**寻址
- 协议：IP、ICMP、ARP、OSPF、RIP、BGP
- 设备：**路由器(Router)**
- **无连接**服务（IP数据报方式）

#### 传输层（第4层）

- 实现**进程到进程**（端到端）通信
- **核心功能**：
  - 复用(发送端)与分用(接收端)
  - 差错检测
  - 提供可靠传输(TCP)或不可靠传输(UDP)
- 通过**端口号(Port)**标识应用进程
- 数据单位：**报文段(Segment)** —— TCP
- 协议：TCP、UDP

#### 会话层（第5层）—— OSI独有

- 建立、管理、释放会话连接（两应用进程间的对话）
- 实现**同步控制**：插入检查点，出错后重传从检查点之后的数据而非全部重传
- ★ 408考点：会话层允许在两个实体间建立会话，并管理和同步会话

#### 表示层（第6层）—— OSI独有

- **数据格式转换**：将应用层数据转换为人可读格式
- **数据压缩**：减少传输数据量
- **数据加密/解密**：保证数据安全性
- ★ 408考点：表示层处理通信双方交换信息的**语法和语义**

#### 应用层（第7层）

- 为用户应用进程提供网络服务接口
- 典型协议：HTTP、FTP、DNS、SMTP

### 1.3.3 OSI模型数据封装与解封装

```
发送方封装过程（自上而下）：

应用层报文
   ↓ + 应用层头部
表示层 PDU
   ↓ + 表示层头部
会话层 PDU
   ↓ + 会话层头部
传输层报文段 ←  + 传输层头部(H4)
   ↓ + 网络层头部(H3)
网络层分组(IP数据报)
   ↓ + 数据链路层头部(H2) + 尾部(T2)
数据链路层帧
   ↓
物理层比特流

接收方按相反路径逐层解封装
```

## 1.5 OSI与TCP/IP模型的对比 ——★408常考简答

| 对比维度 | OSI参考模型 | TCP/IP模型 |
|---------|-----------|-----------|
| **层次数** | 7层 | 4层 |
| **核心思想** | 先有模型，后有协议（理论驱动） | 先有协议，后有模型（实践驱动） |
| **首次提出** | 1984年(ISO) | 1970s(ARPANET) |
| **层次划分** | 会话层、表示层独立 | 会话层/表示层并入应用层 |
| **面向连接 vs 无连接** | 网络层支持两种 | 网络层仅无连接(IP) |
| **服务/协议/接口** | 严格区分三者 | 未严格区分 |
| **实用性** | 理论完善但复杂 | 实际运行广泛（因特网基础） |
| **通用性** | 通用框架 | 特定于TCP/IP协议栈 |

> ★ **记忆口诀**：OSI七层——"物数网传会表应" / TCP/IP四层——"（应用传网接）应用传输网际网络接口"



## 1.6 服务与协议的关系 ——★408概念辨析

| 概念 | 协议 | 服务 |
|------|------|------|
| **方向** | **水平**方向（对等实体间） | **垂直**方向（下层→上层） |
| **可见性** | 对上层透明（上层看不到协议实现） | 上层通过SAP可见（接口可见） |
| **关系** | 协议的实现保证了服务 | 服务是协议要实现的功能 |
| **举例** | TCP如何实现可靠传输 | TCP提供可靠的端到端传输服务 |

> ★★★ 核心结论：**协议是"水平的"，服务是"垂直的"**。



## 1.7 ★★★ 408真题核心考点归纳

### 知识点1：时延计算（必考，选择题/综合题）

**公式集合**：
$$T_{总} = \frac{L}{R} + \frac{D}{V} + T_{处理} + T_{排队}$$

其中 $L$ = 数据长度(bit)，$R$ = 发送速率(bps)，$D$ = 物理距离(m)，$V$ = 传播速率(m/s)

**典型陷阱**：
- 发送时延与传播时延的**定义混淆**（前者在机器内，后者在线路上）
- RTT计算中**是否包含发送时延**（通常不包含）
- 多个分组发送时，**流水线(流水线传输)**下总时间的计算

### 知识点2：OSI七层模型层次与功能匹配（必考，选择题）

- 给一个功能描述，判断属于哪一层
  - "将数据转换为标准格式（编码/压缩/加密）" → 表示层
  - "实现主机到主机通信" → 网络层
  - "可靠端到端传输" → 传输层
  - "差错控制" → 数据链路层（点对点）/ 传输层（端到端）
- OSI模型中路由选择发生在→**网络层**
- OSI模型中数据加密和压缩发生在→**表示层**

### 知识点3：协议三要素匹配（常考，选择题）

- 给一句描述判断属于语法/语义/同步
  - 格式定义 → **语法**
  - 含义定义 → **语义**
  - 时序定义 → **同步**

### 知识点4：五层PDU对应关系（常考，选择题）

| 层 | PDU名称 |
|----|--------|
| 物理层 | 比特(Bit) |
| 数据链路层 | 帧(Frame) |
| 网络层 | 分组/数据报(Packet/Datagram) |
| 传输层 | 报文段(TCP) / 用户数据报(UDP) |
| 应用层 | 报文(Message) |

> 注意：408中网络层PDU统一称为**分组**，TCP称为**报文段**，UDP称为**用户数据报**。

### 知识点5：数据在各层封装的头部变化（常考，选择题）

- 数据链路层：加帧头 + 帧尾
- 网络层：加IP头
- 传输层：加TCP/UDP头
- 物理层：不加任何头部

## 1.8 408真题典型题型示例

### 【例1】时延计算

> 主机甲通过1个路由器(存储转发)与主机乙相连，链路速率均为R bps，传输距离均为D km，信号传播速率为V km/s，甲发送一个长度为L bit的分组给乙。计算：从甲开始发送到乙全部接收的总时延。

**思路**：
$$T_{总} = (发送时延_甲 + 传播时延_{甲→路由}) + [(发送时延_{路由}) + 传播时延_{路由→乙}]$$
$$= \frac{L}{R} + \frac{D}{V} + \frac{L}{R} + \frac{D}{V}$$
$$= \frac{2L}{R} + \frac{2D}{V}$$

**注意**：路由器需**先完整接收再转发**（存储转发），所以有两个发送时延。

### 【例2】TCP/IP层与OSI层的对应

> "TCP/IP模型的网际层对应OSI模型的哪一层？"

**答案**：网络层

### 【例3】协议要素判断
