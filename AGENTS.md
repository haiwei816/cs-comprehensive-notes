# AGENTS.md —— 技术笔记编写规范

> 适用于 Obsidian，所有技术学科的笔记按此规范编写

---

## 1. 目录结构

```
项目根目录/
├── AGENTS.md                          # 本文件（通用规范）
├── 〇〇知识索引.md                      # 学科总索引，使用 [[wikilink]] 链接各章
└── SubjectName/                       # 学科专用英文文件夹
    ├── 第X章_章节中文名.md              # 章节笔记
    └── ...
```

示例：
```
计算机综合/
├── AGENTS.md
├── 计算机网络知识索引.md
├── ComputerNetworking/
│   └── 第一章_计算机网络体系结构.md
└── LuaLearning/
    ├── Lua知识点索引.md
    └── ...
```

---

## 2. 文件命名

- **总索引文件**：`中文名索引.md`（位于项目根目录）
- **章节文件**：`第X章_中文名称.md`（位于对应学科文件夹）
- **学科文件夹**：使用英文命名，如 `ComputerNetworking`、`LuaLearning`

---

## 3. 双向链接（Obsidian wikilink）

### 3.1 索引 → 章节

使用 Obsidian 标题原生 wikilink，不添加 `^块ID` 或 HTML 锚点：

```markdown
[[第一章_计算机网络体系结构#1.1 计算机网络概述|计算机网络概述]]
```

格式：`[[文件名#标题文本|显示文本]]`

| 部分 | 说明 |
|------|------|
| `文件名` | 目标章节文件名（不含路径，Obsidian 全局匹配） |
| `#标题文本` | 章节文件中 `##` 或 `###` 的标题原文，不含标记符号 |
| `\|显示文本` | 链接显示的文字（别名） |

### 3.2 章节 → 索引

每个章节文件顶部添加返回链接：

```markdown
> 🔙 [[计算机网络知识索引|返回索引]]
```

### 3.3 子条目链接

索引中每个可跳转的条目都写成链接，嵌套时只嵌套**内容条目**（链接在外层）：

```markdown
- [[第一章_计算机网络体系结构#1.1.5 计算机网络的性能指标|性能指标]]
  - 速率、带宽、吞吐量
  - 时延计算
```

### 3.4 跨学科链接

不同学科之间也可用 wikilink 互相引用：

```markdown
- C 语言中实现协程的思路可参考 [[Lua知识点索引#第9章 协同程序（Coroutine）|Lua 协程]]
```

## 4. 章节文件结构

### 4.1 文件头部

```markdown
# 第X章  中文名称

> 参考：×××《参考书1》、×××《参考书2》
> 🔙 [[〇〇知识索引|返回索引]]

---
```

### 4.2 标题层级

| 标记 | 层级 | 用途 |
|------|------|------|
| `#` | 一级标题 | 章节主标题（整个文件仅一个） |
| `##` | 二级标题 | 小节标题（如 `## 1.1 概述`） |
| `###` | 三级标题 | 细节目录（如 `### 1.1.1 定义`） |
| `####` | 四级标题 | 进一步细分 |

**标题必须使用 `##` / `###` 纯 Markdown 语法，严禁使用 HTML `<h2>`/`<h3>`。**

### 4.3 重要性标注

用 ★ 标注重要程度（放在标题或行尾）：

| 标记 | 含义 |
|------|------|
| ★★★ | 核心 / 必会 / 高频考点 |
| ★★ | 重要 / 常用 |
| ★ | 了解 / 进阶 |

```markdown
## 1.3 OSI七层参考模型 ——★★★ 核心考点
```

## 5. 内容编写规范

### 5.1 知识点整理

- 每个知识点先给出**定义或结论**，再展开说明
- **关键术语**用 `**加粗**` 标出，便于 skim
- 对比内容用**表格**呈现
- 示例代码用 \`\`\` 代码块

### 5.2 公式

使用 LaTeX 语法：

```markdown
$$T_{send} = \frac{L}{R}$$
```

行内公式：`$E = mc^2$`

### 5.3 重点提示

```markdown
> ★★★ **核心结论**：协议是"水平的"，服务是"垂直的"。
```

### 5.4 表格

保持对齐清晰：

```markdown
| 类型 | 公式 | 说明 |
|------|------|------|
| **发送时延** | $$T = L/R$$ | 数据帧长、带宽 |
```

### 5.5 代码块

标注语言以启用语法高亮：

````markdown
```lua
function hello()
    print("Hello, World!")
end
```
````

## 6. 索引文件规范

索引采用层级列表，每个可跳转条目包裹在 wikilink 中：

```markdown
# 〇〇知识点索引

> 参考：...
> 🔙 [返回项目总索引 →](./总索引.md)

---

## 第一篇  标题

### 第1章 [[文件名#标题|显示名]]
- [[文件名#子标题1|子条目1]]
- [[文件名#子标题2|子条目2]]

## 第二篇  标题
...
```

**原则**：
- 索引层级与章节文件标题层级对应
- 每个 `##` / `###` 级别的标题都在索引中有链接
- 标题文本与目标文件中的标题**严格一致**

---

## 7. 生成新笔记流程

1. 创建/确认学科英文文件夹（如 `SubjectName/`）
2. 在文件夹下创建 `第X章_中文名.md`
3. 按 [4.1](#41-文件头部) 写文件头，添加返回索引链接
4. 按 [4.2](#42-标题层级) 写 `##` / `###` 标题层级
5. 在学科索引 `〇〇知识索引.md` 中添加 wikilink 条目
6. 标注重要性（★★★ / ★★ / ★）
7. 索引中标题文本与章节文件中的标题**严格一致**

---

## 8. 禁止事项

| 禁止 | 正确做法 |
|------|---------|
| ❌ HTML 锚点 `<h2 id="...">` `<div id="...">` `<a id="...">` | 纯 Markdown `##` 标题 |
| ❌ `^块ID` 语法 | Obsidian 原生标题链接 `#标题文本` |
| ❌ 链接中 `#标题` 与目标标题不一致 | 严格一致（含空格、标点） |
| ❌ 索引中 wikilink 嵌套 | 链接在外层，内容细节内嵌 |
| ❌ 混用不同链接语法 | 统一使用 `[[wikilink]]` |
